# План реализации ChatList

Поэтапный порядок действий по реализации программы на основании [PROJECT.md](PROJECT.md).

---

## Этап 1. Подготовка проекта

1. Создать структуру каталогов:
   - `main.py` — точка входа и GUI
   - `db.py` — работа с SQLite
   - `models.py` — логика нейросетей
   - `network.py` — HTTP-запросы к API
   - `.env.example` — шаблон переменных окружения для API-ключей
2. Обновить `requirements.txt`: PyQt6, httpx (или requests), python-dotenv.
3. Добавить в `.gitignore` файл `.env` (ключи не попадают в репозиторий).
4. Описать схему БД в [DATABASE.md](DATABASE.md) и согласовать её с модулями.

---

## Этап 2. База данных (`db.py`)

1. Реализовать подключение к SQLite (один файл, например `chatlist.db`).
2. Написать функцию инициализации: создание таблиц `prompts`, `models`, `results`, `settings` по схеме из DATABASE.md.
3. Реализовать CRUD для **prompts**:
   - добавление промта (дата, текст, теги);
   - получение списка с поиском и сортировкой;
   - выбор одного промта по id.
4. Реализовать CRUD для **models**:
   - добавление / редактирование / удаление модели;
   - получение списка активных (`is_active = 1`);
   - переключение `is_active`.
5. Реализовать операции для **results**:
   - пакетная вставка выбранных строк;
   - чтение с поиском и сортировкой.
6. Реализовать **settings**: чтение и запись пар ключ–значение.
7. Добавить начальные данные (seed): несколько типовых моделей и настройки по умолчанию.

---

## Этап 3. Сеть и модели (`network.py`, `models.py`)

1. В `network.py` реализовать базовый клиент HTTP (таймауты, обработка ошибок, логирование).
2. В `models.py` описать структуру данных модели (имя, URL, id переменной ключа, тип провайдера).
3. Реализовать загрузку API-ключей из `.env` по полю `api_id` таблицы `models`.
4. Реализовать адаптеры под разные типы API (минимум: OpenAI-совместимый формат; опционально: DeepSeek, Groq).
5. Реализовать функцию `send_prompt(prompt_text, active_models)`:
   - параллельная или последовательная отправка в каждую активную модель;
   - возврат списка `{model_name, response_text, error?}` для временной таблицы.
6. Обработать сценарии: нет ключа в `.env`, сеть недоступна, ошибка API — показывать текст ошибки в ячейке ответа.

---

## Этап 4. Временная таблица результатов (в памяти)

1. Определить структуру строки в памяти: `model_name`, `response_text`, `selected` (bool), `model_id`.
2. При новом промте — полностью очищать временную таблицу.
3. После получения ответов от API — заполнять временную таблицу.
4. При нажатии «Сохранить» — передавать в `db.py` только строки с `selected = True`, затем очищать временную таблицу.

---

## Этап 5. Графический интерфейс (`main.py`)

### 5.1. Общая компоновка

1. Главное окно: область ввода/выбора промта, кнопка отправки, таблица результатов, кнопки действий.
2. Единый стиль (тёмная или светлая тема), согласованный с текущим прототипом.

### 5.2. Работа с промтами

1. Поле ввода нового промта (многострочное).
2. Выпадающий список или панель сохранённых промтов из БД.
3. При выборе сохранённого — подставлять текст в поле ввода.
4. Кнопка «Отправить» — запускает этап 4.2 из PROJECT.md.

### 5.3. Таблица результатов

1. Колонки: чекбокс, название модели, текст ответа.
2. Чекбокс `selected` на каждой строке.
3. Кнопка «Сохранить выбранные» — запись в `results`.
4. Индикатор загрузки на время ожидания ответов от API.

### 5.4. Управление данными (вкладки или отдельные окна)

1. **Промты** — таблица с поиском и сортировкой по дате, тексту, тегам.
2. **Модели** — список нейросетей: имя, URL, переменная ключа, активна/неактивна; форма добавления и редактирования.
3. **Результаты** — сохранённые ответы с поиском и сортировкой.
4. **Настройки** — параметры программы из таблицы `settings`.

## Этап 6. Связка модулей и сценарии

1. Подключить `db.py`, `models.py`, `network.py` к `main.py`.
2. Прогнать основной сценарий:
   - ввод промта → отправка → отображение → выбор строк → сохранение;
   - новый промт → временная таблица сбрасывается;
   - выбор промта из БД → отправка.
3. Проверить граничные случаи: нет активных моделей, пустой промт, ни одна строка не отмечена при сохранении.

```
Этап 1 → Этап 2 → Этап 3 → Этап 4 → Этап 5 → Этап 6 → Этап 7 → Этап 8
```
