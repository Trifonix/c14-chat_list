# Спецификация ChatList

## 1. Назначение

**ChatList** — Python-приложение с графическим интерфейсом (PyQt6), которое отправляет один промт в несколько нейросетей параллельно и позволяет сравнить ответы. Пользователь отмечает нужные строки и сохраняет их в постоянную базу SQLite.

## 2. Стек

## 4. База данных

Таблицы SQLite (подробнее — [DATABASE.md](DATABASE.md)):

- **prompts** — сохранённые промты (дата, текст, теги)
- **models** — нейросети (имя, URL, переменная `.env`, активность, тип)
- **results** — сохранённые ответы (связь prompt + model)
- **settings** — настройки приложения (ключ–значение)
- **request_logs** — журнал HTTP-запросов к API

API-ключи в БД **не хранятся** — только имя переменной (`api_id`), значение читается из `config/.env`.

## 5. Рабочий процесс

### 5.1 Вкладка «Запрос»

1. Пользователь вводит промт или выбирает сохранённый из выпадающего списка.
2. По кнопке **Отправить** промт уходит во все активные модели (`is_active = 1`).
3. Ответы появляются во временной таблице (в памяти): чекбокс, модель, текст, кнопка **Открыть** (Markdown).
4. **Сохранить выбранные** — запись отмеченных строк в `results` (промт при необходимости создаётся в `prompts`).
5. **Очистить результаты** — сброс временной таблицы.
6. **Экспорт MD / JSON** — выгрузка текущей сессии в файл.

При изменении текста промта или новом запросе временная таблица очищается.

### 5.2 Остальные вкладки

| Вкладка | Функции |
|---------|---------|
| **Промты** | Таблица промтов, поиск, сортировка, удаление |
| **Модели** | Список моделей + форма редактирования, вкл/выкл, удаление |
| **Результаты** | Сохранённые ответы, поиск, экспорт, удаление |
| **Логи** | Журнал запросов к API, поиск, очистка |
| **Настройки** | Таймаут, тема, теги по умолчанию, путь к БД |

## 6. Поддерживаемые типы моделей

OpenAI-совместимый формат chat/completions:

- `openrouter`
- `openai`
- `deepseek`
- `groq`

Отправка параллельная (ThreadPoolExecutor), порядок результатов сохраняется по списку моделей.

## 7. Обработка ошибок

- Нет ключа в `.env` — текст ошибки в ячейке ответа
- HTTP-ошибка, таймаут, некорректный JSON — сообщение в ответе и запись в `request_logs`
- Пустой промт / нет активных моделей / нет выбранных строк — диалог-предупреждение

## 10. Функция "AI-ассистент для улучшения промтов" — ✅ реализовано

### 10.1. Принимает исходный текст промта от пользователя.
### 10.2. Отправляет его в одну указанную модель (через существующий OpenRouter-клиент).
### 10.3. Возвращает:
   - улучшенную версию промта;
   - 2–3 варианта переформулировки;
   - при необходимости — адаптацию промта под разные типы моделей (код, анализ, креатив).
### 10.4. Встраивает это в UI:
   - отдельная кнопка «Улучшить промт» рядом с полем ввода;
   - окно/панель, где пользователь видит:
     -- исходный промт;
     -- улучшенный;
     -- альтернативные варианты;
   - возможность «Подставить в поле ввода» выбранный вариант одним кликом.

Модуль: `src/prompt_improver.py`. Модель ассистента настраивается в **Настройки** (`prompt_improver_model_id`).
