## Этап 9. AI-ассистент для улучшения промтов (п. 10) — ⬜

### 9.1. Бэкенд (`src/prompt_improver.py`)

1. Создать модуль `src/prompt_improver.py` (не смешивать с `src/models.py` — там отправка в несколько моделей для сравнения).
2. Описать dataclass результата, например `PromptImprovementResult`:
   - `original` — исходный текст;
   - `improved` — улучшенная версия;
   - `alternatives: list[str]` — 2–3 переформулировки;
   - `model_hints: dict[str, str]` — опционально: подсказки под тип задачи (`code`, `analysis`, `creative`).
3. Реализовать функцию `improve_prompt(text: str, model: AiModel, timeout: float, db: Database | None) -> PromptImprovementResult | str`:
   - один вызов существующего `call_model()` из `src/models.py`;
   - системный промт с инструкцией вернуть **структурированный JSON** (удобнее парсить, чем свободный текст);
   - при ошибке API — вернуть строку с текстом ошибки (как в основном потоке).
4. Добавить парсер ответа: `json.loads` + fallback на regex/эвристику, если модель вернула markdown-блок `` ```json ``.
5. Запись в `request_logs` через уже существующий механизм в `call_model()`.

**Пример системного промта (черновик):**

> Ты помощник по улучшению промтов. Верни JSON: `{"improved": "...", "alternatives": ["...", "..."], "model_hints": {"code": "...", "analysis": "...", "creative": "..."}}`. Язык ответа — как у исходного промта.

### 9.2. Настройка модели для ассистента

1. В таблицу `settings` добавить ключ `prompt_improver_model_id` (id из `models`, по умолчанию — первая активная OpenRouter-модель).
2. На вкладке **Настройки** (см. этап 10) — выпадающий список «Модель для улучшения промта» (только активные модели).
3. Seed в `src/db.py`: значение по умолчанию при первом запуске.

### 9.3. GUI — вкладка «Запрос» (`main.py`)

1. Кнопка **«Улучшить промт»** рядом с **«Отправить»** (в блоке действий над/под полем ввода).
2. Валидация: пустой промт → `QMessageBox.warning`, как при отправке.
3. Фоновый поток `ImprovePromptWorker(QThread)` по аналогии с `SendPromptWorker`:
   - сигналы `finished(PromptImprovementResult)`, `failed(str)`, `status(str)` для строки статуса.
4. На время работы — блокировать кнопки «Отправить» / «Улучшить промт», показывать статус «Улучшение промта…».

### 9.4. GUI — диалог результата

1. Класс `PromptImprovementDialog(QDialog)`:
   - секция **Исходный промт** (только чтение);
   - секция **Улучшенный** + кнопка «Подставить»;
   - список **Альтернативы** (2–3) — каждая с кнопкой «Подставить»;
   - опционально — раскрывающийся блок **Под разные задачи** (code / analysis / creative), если `model_hints` не пуст;
   - кнопка **Закрыть**.
2. «Подставить» — `prompt_edit.setPlainText(...)`, закрытие диалога (или оставить открытым — на усмотрение UX; рекомендуется закрывать).
3. Стили диалога — из текущей темы (см. этап 10).

## Этап 10. Настройки: тема и размер шрифта (п. 11) — ⬜

> **Текущее состояние:** вкладка «Настройки» уже сохраняет `theme`, `request_timeout`, `default_tags` в `settings`, но **светлая тема не применяется**, **размер шрифта не реализован**.

### 10.1. Ключи в `settings`

| Ключ | Тип | По умолчанию | Назначение |
|------|-----|--------------|------------|
| `theme` | `dark` / `light` | `dark` | Цветовая схема |
| `ui_font_size` | int (8–24) | `10` | Базовый размер шрифта панелей |
| `prompt_improver_model_id` | int | id первой активной модели | Для этапа 9 |

Seed и миграция в `src/db.py` (`INSERT OR IGNORE`).

### 10.2. Модуль тем (`src/themes.py`)

1. Вынести QSS из `main.py` в `src/themes.py`:
   - `STYLES_DARK: str`
   - `STYLES_LIGHT: str`
   - `def get_stylesheet(theme: str) -> str`
2. Светлая тема — контрастные кнопки, читаемые поля ввода, те же objectName (`#secondaryButton`, `#dangerButton`), что и в тёмной.
3. Функция `def apply_theme(app: QApplication, main_window: QMainWindow, theme: str, font_size: int) -> None`:
   - `app.setFont(QFont("Segoe UI", font_size))`;
   - `main_window.setStyleSheet(get_stylesheet(theme))`;
   - при необходимости — обновить дочерние диалоги при следующем открытии.

### 10.3. Доработка вкладки «Настройки» (`SettingsTab`)

1. Добавить `QSpinBox` «Размер шрифта» (диапазон 8–24, суффикс « pt»).
2. Добавить комбобокс «Модель для улучшения промта» (список из `db.list_models(active_only=True)`).
3. При **Сохранить**:
   - записать все ключи в `settings`;
   - вызвать `apply_theme()` на главном окне **сразу**, без перезапуска.
4. Сигнал или ссылка `SettingsTab → MainWindow` для применения темы (передать callback / `main_window` в конструктор).

### 10.4. Загрузка при старте (`main()`)

1. После `get_database()` прочитать `theme` и `ui_font_size`.
2. Применить тему и шрифт до `window.show()`.
3. Убрать захардкоженный `setStyleSheet(STYLES)` из `MainWindow.__init__` — только через `apply_theme()`.
